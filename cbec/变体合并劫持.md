# 变体合并劫持 (Variation Hijacking / Review Merging)

> [!INFO] Definition
> **变体合并劫持**是指竞争对手将已停产或无人经营的"僵尸 Listing"（拥有大量好评）通过技术手段修改并合并到自己的新产品下，使其瞬间获得大量评论。

## 核心特征

| 维度 | 说明 |
| :--- | :--- |
| **手法** | 寻找僵尸 Listing → 修改为自家产品 → 合并变体 |
| **效果** | 新产品瞬间获得数千条高分评价 |
| **后果** | BSR "弯道超车"，正规产品失去竞争力 |

## 典型场景

例如：一个旧款水杯有 1000 条 5 星好评，已停售 → 竞争对手将其修改为自家耳机产品并合并变体 → 耳机瞬间拥有 1000 条评论

## 应对策略

| 策略 | 说明 |
| :--- | :--- |
| **举报系统** | 举报类目不符 (Category Mismatch) |
| **持续监控** | 使用 Keepa/Helium 10 监控竞品变体数目异常跳变 |
| **品牌保护** | 通过 Brand Registry 监控品牌下所有 ASIN |
